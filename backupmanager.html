Help & installatie
Installatie, systemd service en uitleg
Snelstart — 3 stappen
1
Download & sla op
Sla backupmanager.py en backupmanager.html op in bijv. ~/MjApps/BackUpManager/
2
Start de API server
python3 backupmanager.py --daemon
De CUDA-waarschuwing bovenin is onschadelijk — de server start gewoon.
3
Open de GUI in je browser
firefox backupmanager.html # of: chromium backupmanager.html
Systemd service aanmaken (automatisch opstarten)
Stap 1 — Maak de map aan als die nog niet bestaat:
mkdir -p ~/.config/systemd/user
Stap 2 — Maak het service-bestand aan met een teksteditor:
nano ~/.config/systemd/user/backupmanager.service # of: gedit ~/.config/systemd/user/backupmanager.service # of: kate ~/.config/systemd/user/backupmanager.service
Stap 3 — Plak deze inhoud in het bestand (pas het pad aan naar jouw locatie):
[Unit] Description=BackupManager API After=network.target [Service] ExecStart=python3 /home/mjeppo/MjApps/BackUpManager/backupmanager.py --daemon Restart=on-failure RestartSec=5 [Install] WantedBy=default.target
Stap 4 — Sla op en activeer de service:
systemctl --user daemon-reload systemctl --user enable --now backupmanager systemctl --user status backupmanager # controleer of hij draait
Service stoppen of herstarten:
systemctl --user stop backupmanager systemctl --user restart backupmanager
Uitsluitingen — glob-patronen
In het veld 'Uitsluitingen' kun je glob-patronen opgeven. Voorbeelden:
*.tmp → alle .tmp bestanden / all .tmp files *.log → alle logbestanden / all log files .git → de .git map / the .git directory node_modules → de node_modules map __pycache__ → Python cache Thumbs.db → Windows thumbnails /pad/naar/map → specifieke submap / specific subfolder
